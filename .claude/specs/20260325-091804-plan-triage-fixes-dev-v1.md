## Implementation Phases

### Phase 1: P1 Fixes (High Priority)

#### Fix 1: Add CloudWatch Logs permissions to deploy role

The `eagle-deploy-role-dev` IAM role needs these permissions:
- `logs:DescribeLogGroups`
- `logs:StartQuery`
- `logs:GetQueryResults`
- `logs:GetLogEvents`
- `logs:FilterLogEvents`

#### Fix 2: Improve SSO token error handling in Strands agent

The Strands agent should detect `TokenRetrievalError` and surface it clearly rather than producing null-output traces.

### Phase 2: P2 Improvements (Medium)

#### Improvement 1: Add session_id to all Langfuse traces
Currently error traces have `session_id: null`, preventing cross-referencing. Ensure all agent invocations pass session_id to Langfuse.

#### Improvement 2: Re-run triage after fixes
Validate that CloudWatch is now accessible and error rate drops.

## Step by Step Tasks

### 1. Add CloudWatch Logs permissions to CI/CD deploy role

- **File**: `infrastructure/cdk-eagle/lib/ci-cd-stack.ts` (or whichever stack defines `eagle-deploy-role-dev`)
- **Problem**: The OIDC-assumed role has ECS/CDK deploy permissions but no CloudWatch Logs read access. This blocks CI triage from querying `/eagle/ecs/backend-dev`, `/eagle/ecs/frontend-dev`, `/eagle/app`.
- **Fix**: Add an IAM policy statement granting `logs:DescribeLogGroups`, `logs:StartQuery`, `logs:GetQueryResults`, `logs:GetLogEvents`, `logs:FilterLogEvents` on `arn:aws:logs:us-east-1:695681773636:log-group:/eagle/*`.
- **Validation**:
  - `npx cdk synth --quiet` — CDK compiles
  - Deploy stack, then verify: `aws logs describe-log-groups --log-group-name-prefix /eagle/ecs/backend-dev`

### 2. Add SSO token pre-check to Strands agent invocation

- **File**: `server/app/strands_agentic_service.py`
- **Problem**: When SSO token expires, `strands.Agent` raises `TokenRetrievalError` but it's caught silently, producing null-output Langfuse traces.
- **Fix**: Wrap the agent invocation in a try/except that catches `botocore.exceptions.TokenRetrievalError` and raises a clear `HTTPException(503, "Bedrock credentials expired")` or returns an SSE error event.
- **Validation**:
  - `ruff check app/` — lint passes
  - `python -m pytest tests/ -v` — unit tests pass

### 3. Ensure session_id is passed to Langfuse traces

- **File**: `server/app/strands_agentic_service.py` and/or `server/app/streaming_routes.py`
- **Problem**: 12/12 error traces had `session_id: null`, preventing cross-reference with DynamoDB feedback.
- **Fix**: Pass the session_id from the chat request through to the Langfuse trace creation call.
- **Validation**:
  - Send a test message and verify the Langfuse trace has `sessionId` populated

### 4. Validate All Fixes

- `npx tsc --noEmit`
- `python -m pytest tests/ -v`

- Re-run `/triage light` to confirm CloudWatch is now accessible and error count drops
