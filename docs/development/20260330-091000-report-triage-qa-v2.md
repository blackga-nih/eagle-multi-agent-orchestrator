### Error Pattern Clusters

#### Cluster 1: Orphan Stream Traces (7 occurrences)
- **Langfuse signal**: Traces named `eagle-stream-*` with no session_id, no user_id, 0 latency, 0 cost, null output
- **Category**: Instrumentation artifact — Langfuse traces are created at stream initialization but never populated with observations or output
- **Severity**: Warning — inflates error metrics but does not indicate user-facing failures
- **Likely cause**: `MultiAgentStreamWriter` or SSE streaming code creates a Langfuse trace at connection time, but if the request fails early or is a health check, the trace remains empty

#### Cluster 2: Silent Request Failure (1 occurrence)
- **Langfuse signal**: Real user session (dev-user, session 114df9de) with 136ms latency, $0 cost, null output
- **Category**: Auth/infra failure — request reached the backend but no model invocation occurred
- **Severity**: P2 — single occurrence, no user feedback, but indicates a code path where requests fail silently
- **Likely cause**: Bedrock model invocation failed (auth, throttling, or config issue) but error was swallowed

#### Cluster 3: CI IAM Permission Gap (persistent)
- **CloudWatch signal**: All 3 log groups return AccessDeniedException for `eagle-deploy-role-dev`
- **Category**: IAM configuration gap
- **Severity**: P1 — blocks automated triage, reduces observability in CI pipeline
- **Likely cause**: The CDK CI/CD stack does not grant CloudWatch Logs read permissions to the deploy role
