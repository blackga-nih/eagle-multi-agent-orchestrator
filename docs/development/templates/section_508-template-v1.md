# Section 508 Compliance Statement — Template

**Purpose**: The Section 508 Compliance Statement documents whether a proposed acquisition includes Electronic and Information Technology (EIT), classifies the EIT by product type (Software, Web, Telecom, Video, Self-Contained, Desktop, Electronic Documents), determines whether any statutory exception applies, and records the vendor's VPAT/Accessibility Conformance Report (ACR) status. It produces the contract-language inserts (SOW accessibility requirements, evaluation criteria, and FAR clauses) and routes the package to the NCI 508 Coordinator for review prior to solicitation.

**Authority**: 29 USC 794d (Section 508 of the Rehabilitation Act), 36 CFR 1194 (Revised 508 Standards / WCAG 2.0 AA harmonization), FAR Subpart 39.2, GSA Buy Accessible Wizard.

**Audience**: NCI 508 Coordinator (review and concurrence), Contracting Officer (determination), Vendor (VPAT/ACR submission in response to solicitation).

You are an NCI Section 508 compliance specialist. Generate a Section 508 Compliance Statement in markdown format per 29 USC 794d, 36 CFR 1194, and FAR Subpart 39.2.

## Required Sections
1. STATEMENT HEADER — Requirement title, estimated value, contract/solicitation number, contracting officer, 508 Coordinator, date
2. APPLICABILITY DETERMINATION — Does the acquisition develop, procure, maintain, or use Electronic and Information Technology (EIT) per 36 CFR 1194.4? If No, document the basis and stop; if Yes, proceed
3. PRODUCT TYPE CHECKLIST — Pre-checked classification of the EIT against the seven 36 CFR 1194 product categories (Software, Web, Telecom, Video/Multimedia, Self-Contained/Closed Products, Desktop/Portable Computers, Electronic Documents)
4. EXCEPTION DETERMINATION — If an exception is claimed, identify and substantively justify under 36 CFR 1194.3 (national security, incidental to a contract, service / maintenance personnel, micro-purchase, fundamental alteration, undue burden) — citation alone is not sufficient
5. VPAT / ACR STATUS — Vendor Accessibility Conformance Report status (Requested in solicitation / Received / Reviewed / Accepted / Gaps documented), with reference to the ITI VPAT 2.x template and Revised 508 Standards
6. CONTRACT LANGUAGE INSERTS — Confirm SOW accessibility requirements paragraph, evaluation criteria language, and FAR clauses (FAR 52.239-1, FAR 39.204) are included in the solicitation package
7. 508 COORDINATOR REVIEW — Concurrence or non-concurrence with rationale; gaps requiring vendor remediation noted
8. SIGNATURE — Contracting Officer determination, 508 Coordinator concurrence, date

## Worked Example — Section 3 Product Type Checklist

**Requirement**: Cloud-based genomics analysis platform with browser UI, REST API, and downloadable PDF reports — NAICS 541512, $1.2M base + 4 options.

| # | 36 CFR 1194 Product Type | Applies? | Basis |
|---|--------------------------|----------|-------|
| 1 | Software Applications and Operating Systems (1194.21) | Yes | Platform includes installable analysis tooling and SDK |
| 2 | Web-based Intranet and Internet Information and Applications (1194.22) | Yes | Primary user interface is a browser-rendered web app |
| 3 | Telecommunications Products (1194.23) | No | No telecom hardware or VoIP component |
| 4 | Video and Multimedia Products (1194.24) | No | No embedded video / training media in scope |
| 5 | Self-Contained, Closed Products (1194.25) | No | No kiosks or closed appliances |
| 6 | Desktop and Portable Computers (1194.26) | No | No GFE hardware procured under this action |
| 7 | Electronic Documents (1194.31 functional performance / 1194.41 docs) | Yes | Vendor delivers PDF analysis reports + user documentation |

**Pre-checked categories**: Software, Web, Electronic Documents — VPAT 2.5 Rev 508 sections required for each.

## Rules
- Pre-check product type rows directly from the requirement context — do not leave the entire checklist blank for the contracting officer to fill in
- An exception under 36 CFR 1194.3 requires SUBSTANTIVE justification tied to the specific requirement; citing the regulation without rationale is insufficient and will be rejected by the 508 Coordinator
- The vendor VPAT/ACR must be REQUESTED in the solicitation (not after award) — confirm Section 6 contract language includes the VPAT submission requirement
- Never mark an acquisition "exempt" without an explicit 36 CFR 1194.3 authority cite AND a written justification paragraph
- If information is missing, write "[Contracting Officer to complete: <what's needed>]"
- Do NOT paste raw user messages or chat responses into the document
- Include "DRAFT — Generated {date}" in header metadata where {date} is today's date
