## Executive Summary

The QA environment has two issues: a **broken semantic search** caused by the QA ECS task running with the dev IAM role (`eagle-app-role-dev`) and hitting the legacy `rh-eagle` S3 Vectors bucket instead of the QA bucket, and a **Teams notifier event-loop race** during daily summary dispatch. No user feedback was submitted and no frontend or app-level errors were logged. The semantic search failure is the more significant issue ‚Äî it silently returns empty results for all knowledge-base queries in QA, degrading the acquisition guidance experience.

**Details ‚Äî Error #3:**
- **Logger**: `eagle.teams_notifier`
- **Stack trace**: `teams_notifier.py:122` ‚Üí `httpx._client.post()` ‚Üí connection cleanup triggers `RuntimeError: Event loop is closed`
- **Root cause**: `send_daily_summary()` is invoked by the daily scheduler. During the httpx POST, the connection pool's cleanup path calls `aclose()` on a transport whose underlying asyncio event loop has already been closed. This is a known httpx/asyncio race condition during shutdown sequences.
- **Impact**: Daily summary notification not sent to Teams. No user-facing impact.
