## Executive Summary

The QA environment has a **P0 Bedrock IAM permission gap** causing all semantic search embedding requests to fail. The `eagle-app-role-dev` IAM role lacks `bedrock:InvokeModel` permission for `amazon.titan-embed-text-v2:0`, resulting in `AccessDeniedException` on every `embed_text()` call. This was triggered by an active user (2 sessions on 2026-04-27) performing procurement knowledge searches. Semantic search silently degrades to path-based search only, reducing knowledge retrieval quality for all QA users. The fix is a one-line addition to the CDK core-stack IAM policy. Infrastructure gaps from the prior triage (CI hook path, Langfuse QA credentials, frontend-qa logging) remain unresolved.
