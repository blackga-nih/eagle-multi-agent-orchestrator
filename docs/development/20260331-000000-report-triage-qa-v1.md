## Executive Summary

Two distinct infrastructure issues affect the QA environment: (1) the OpenTelemetry OTLP span exporter is receiving **401 Unauthorized** responses when shipping traces to Langfuse, causing observability data loss; and (2) **AWS SSO token expiration** is causing Bedrock agent invocations to fail silently, blocking all user queries until SSO is manually refreshed. No user feedback exists for the `default-qa` tenant, so these issues were detected only through backend telemetry. Zero users appear to be actively using QA under the `default-qa` tenant in this window.

All report `TokenRetrievalError: Error when retrieving token from sso: Token has expired and refresh failed`. These are Bedrock `invoke_agent` calls that fail immediately (latency <2ms, cost $0) because the boto3 credential chain cannot refresh the SSO token. **Orphan Query Traces** (5 traces):
Named `eagle-query-{session_prefix}` with no userId, no sessionId, 0ms latency, $0 cost. These are wrapper traces created before the agent invocation that were never completed — a side effect of the SSO failure causing the parent operation to abort before the wrapper trace could be populated.
