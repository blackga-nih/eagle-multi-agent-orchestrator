## Executive Summary

Knowledge search semantic lane is broken in QA due to an IAM permission gap. The shared `eagle-app-role-dev` role lacks `s3vectors:QueryVectors` for the QA vectors bucket (`eagle-kb-vectors-695681773636-qa`). This causes the semantic search to silently degrade — falling back to metadata/path-only lanes. No user feedback was submitted in the 24h window, but any QA user performing knowledge searches would get reduced-quality results.

**Causal chain**: The two errors are correlated by timing (5 seconds apart, same task). The sequence is:
1. `exec_semantic_search` calls S3 Vectors `QueryVectors` → `AccessDeniedException` (semantic lane fails)
2. System falls back to AI ranking path → LLM returns malformed JSON → fallback also fails
3. Net result: knowledge search returns empty/degraded results

### Root Cause Analysis

The QA Compute Stack (`EagleComputeStackQA`) shares `core.appRole` from the dev Core Stack (`EagleCoreStack`). The Core Stack is instantiated with `DEV_CONFIG`, so the S3 Vectors IAM policy only grants access to:

```
arn:aws:s3vectors:us-east-1:695681773636:bucket/eagle-kb-vectors-695681773636-dev
arn:aws:s3vectors:us-east-1:695681773636:bucket/eagle-kb-vectors-695681773636-dev/index/eagle-kb-approved
```

But the QA backend needs access to:

```
arn:aws:s3vectors:us-east-1:695681773636:bucket/eagle-kb-vectors-695681773636-qa
arn:aws:s3vectors:us-east-1:695681773636:bucket/eagle-kb-vectors-695681773636-qa/index/eagle-kb-approved
```

**Architecture decision** (line 110 of `bin/eagle.ts`): `appRole: core.appRole` — QA intentionally shares the dev role to avoid IAM role duplication. The policy just needs to cover both buckets.
