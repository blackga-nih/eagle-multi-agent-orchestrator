## Executive Summary

The QA environment has **3 distinct actionable bugs** affecting all active users. The most widespread issue is a **missing IAM permission** for the Titan Embedding model (`amazon.titan-embed-text-v2:0`), which blocks semantic search for every user on every session. Two additional application bugs — a **Float-to-Decimal conversion failure** in document creation and an **argument ordering error** in template resolution — are causing tool failures for the document generation and template management workflows. Three unique users were affected across 7 sessions in the last 24 hours.

| Field | Value |

| Severity | ACTIONABLE — Application Bug |

**Error message**: `TypeError: Float types are not supported. Use Decimal types instead.`

**Stack trace**: `strands_agentic_service.py:3655` → `legacy_dispatch.py:55` → `document_generation.py:562` → `document_service.py:209` → `document_service.py:479 (table.put_item)`

**Root cause**: The `source_data` dict passed to `_create_document_record()` contains Python `float` values (e.g., IGCE `estimated_value` parsed from `"$1.8"`). DynamoDB's boto3 serializer rejects floats — requires `Decimal` types.

#### Error Category 3: Template Resolution Argument Mismatch (ACTIONABLE)

| Field | Value |

| Severity | ACTIONABLE — Application Bug |

**Error message**: `TypeError: resolve_template() got multiple values for argument 'user_id'`

**Root cause**: `admin_tools.py:223` calls `resolve_template(tenant_id, doc_type, user_id=...)` but the function signature is `resolve_template(tenant_id, user_id, doc_type)`. The second positional arg `doc_type` is bound to `user_id`, and then the keyword `user_id` duplicates it.
