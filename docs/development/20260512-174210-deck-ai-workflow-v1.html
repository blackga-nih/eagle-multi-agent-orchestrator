What is EAGLE?

Multi-tenant AI assistant for NCI contracting officers

What it does

  • Answers federal acquisition questions (FAR/DFARS authority)
  • Generates ~10 document types — SOW, IGCE, AP, J&A, EVAL, 508, COR, CTJ, FUND
  • Routes to 9 specialist agents (legal, market intel, policy, compliance, tech, finance, public interest)
  • Tier-gated: basic / advanced / premium

FastAPI · Next.js · Strands SDK (boto3-native Bedrock) · DynamoDB · S3 · ECS Fargate · Cognito

Flywheels — the overnight compounding

5 self-improving loops. Each one means I wake up to diagnoses, not investigations.

  1. Nightly triage — 1 AM PST, cross-source bug discovery
  2. Eval + expert self-improve — 2 AM ET, agent prompt tuning
  3. Morning report — 8 AM ET weekday, Teams Adaptive Card
  4. Post-deploy smoke — on each deploy, 6 scenarios
  5. Expert self-improve — manual, after each ship → cumulative expertise.md
The keystone insight: Each loop's output becomes the next loop's input. Today's commits become tonight's triage corpus, become tomorrow's fix plans.

Flywheel 1: Nightly Triage

cron 0 9 * * * (1 AM PST) — runs against dev + qa matrix

Evidence: 15+ dated triage plans in .claude/specs/ over the past 6 weeks.

Flywheel 2: Eval + Self-Improve

cron 0 6 * * * (2 AM ET) — 142-test eval drives code edits

cron 0 6 * * * (eval.yml) ▼ test_strands_eval.py (142 tests, haiku) ▼ Failure classification by 5 levers: Lever 1: Agent prompts → edit agent.md Lever 2: Skill workflows → edit SKILL.md Lever 3: Supervisor routing → edit supervisor Lever 4: Trigger patterns (YAML) → edit frontmatter Lever 5: Context budget → edit MAX_SKILL_PROMPT_CHARS ▼ /experts:eval:self-improve --fix ▼ (writes CODE, not docs) Re-run affected tests → validate ▼ eval_aws_publisher.py → S3 + Langfuse ▼ expertise.md updated (cumulative)

Closed loop: eval failures → code edits → re-validated → published. Not just a report — actual fixes.

FAQ — Part 1

How is this different from GitHub Copilot?

Copilot is autocomplete inside an editor. This is workflow automation — multi-step skills that read logs, write specs, run tests, generate reports, update Jira. Different layer.

What does this cost us in API spend?

Each triage: ~$0.02-0.05 in Bedrock. Langfuse: free up to 50K traces/mo, then ~$0.0001/trace. Total: under $40/month for the cognitive layer at current pace.

Could non-engineers on the team use this?

For most workflows: type /skill-name in Claude Code. The plan→build→review→ship loop is the only meta-pattern to learn. just registry is the discovery surface for everything else.

FAQ — Part 2

What if Claude Code goes down?

Justfile recipes that wrap actual scripts (langfuse-report, baseline, e2e-judge, deploy, morning-report) work without Claude Code. The print-and-point recipes (triage-session, kb-*, check-*) tell you what to invoke but need Claude. ~60% of recipes work standalone.

How long did this take to build?

~6-8 weeks of incremental work, intermixed with EAGLE feature development. Each expert / skill was added when a recurring task became annoying enough to automate. Not a "drop everything" effort — additive.

What's next?

Real Excalidraw diagrams for each flywheel · QA hardening skill (we identified this gap today) · skill-creator workshop for the team · per-domain skill libraries (legal-counsel could have its own skill family)
