All errors are identical:
```json
{"level": "ERROR", "logger": "opentelemetry.exporter.otlp.proto.http.trace_exporter",
 "msg": "Failed to export span batch code: 401, reason: Unauthorized"}
```

**Root cause analysis**: The backend QA deployment uses `LANGFUSE_PUBLIC_KEY` / `LANGFUSE_SECRET_KEY` from the environment to construct a Basic auth header for the OTLP endpoint (`https://us.cloud.langfuse.com/api/public/otel/v1/traces`). The startup probe (`server/app/strands_agentic_service.py:126-144`) validates auth before registering the exporter. The fact that export errors appear means:
1. The startup probe passed (credentials were valid at boot time)
2. Credentials became invalid mid-flight — likely a Langfuse API key rotation or expiration
3. Error window (15:07–21:27) aligns with active user sessions, confirming spans are generated but not exported

**Impact**: Langfuse trace gaps for QA sessions during this 6-hour window. Observability data is lost.
