## Executive Summary

The QA environment has **1 active P0 bug** affecting all user sessions: the IAM role `eagle-app-role-dev` lacks `bedrock:InvokeModel` permission for `amazon.titan-embed-text-v2:0`, causing semantic search to fail silently for every user. This bug has been **active for 3+ consecutive days** (first reported 2026-04-22) and remains unfixed in the codebase. Today's data shows 5 unique sessions and 1 active user impacted. Three additional bugs (P1 Float→Decimal, P1 resolve_template args, P2 dollar-sign parsing) remain dormant but unfixed — they will resurface when users exercise document generation workflows. The QA environment shares the dev IAM role (`core.appRole`) rather than having an isolated QA-specific role, which means any IAM fix to core-stack.ts will resolve the issue for both environments.
