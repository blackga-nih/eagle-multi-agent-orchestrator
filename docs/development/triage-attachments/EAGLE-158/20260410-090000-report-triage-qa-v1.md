#### Detailed Error Evidence

**Error 1: OTel OTLP 401 Unauthorized (39 occurrences)**

All 39 errors come from `opentelemetry.exporter.otlp.proto.http.trace_exporter` with message `Failed to export span batch code: 401, reason: Unauthorized`. This means every Langfuse span export fails, creating a complete observability blind spot for the QA environment.

Hourly distribution:
| Hour (UTC) | Count |
|------------|-------|
| Apr 9 14:00 | 8 |
| Apr 9 16:00 | 22 |
| Apr 9 20:00 | 9 |

The startup probe in `server/app/strands_agentic_service.py:128-143` is designed to catch 401 errors and skip exporter registration. That it still exports means either: (a) the QA ECS task definition injects different Langfuse keys than `server/.env`, and those keys passed the probe initially but were later rotated/expired; or (b) the probe exception path (`line 145-146`) was triggered, registering the exporter despite an inconclusive probe.

Affected log streams: 2 distinct ECS tasks (`c80576d5cb87...` and `cc39c01b1d26...`), confirming both QA backend containers are affected.

**Errors 2-4: Bedrock ServiceUnavailableException + Circuit Breaker (37 events total)**

The `us.anthropic.claude-sonnet-4-6` model fails Bedrock `Converse` API calls during keepalive pings. The circuit breaker (threshold=2, configured at `strands_agentic_service.py:466`) trips immediately and stays OPEN. Failure count escalated from 2 to 11 on Apr 9, and reset back to 2 on Apr 10 (new container deployment or recovery timeout expiry).
