### Trend Analysis

- **Titan Embed AccessDenied** is a **persistent, recurring issue** — it fires on every semantic search call across all sessions. This has likely been present since the QA environment was last deployed. The CDK code (`core-stack.ts:185-191`) includes the permission, indicating a **deployment gap** — the stack needs to be redeployed, or the QA task is referencing the wrong IAM role.
- **Role mismatch**: The error shows `eagle-app-role-dev` (dev role) being used in the QA backend logs. The CDK creates `eagle-app-role-${config.env}`, so QA should use `eagle-app-role-qa`. This suggests the QA ECS task definition may be pointing to the dev role.
- **Float TypeError** is a **new regression** — the IGCE document flow passes `estimated_value` as a Python float (from Pydantic's `float` type), which DynamoDB rejects. The upstream validation warning shows the dollar-sign input (`$280`) is also not handled.
- **Document timeout** may be related to the same session's embedding failures causing cascading delays, or could be an independent issue with `purchase_request` document generation.
